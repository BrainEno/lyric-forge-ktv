## 补充 8.6 独立播放器模式

### 用户场景
- **场景 D**：用户只想听歌，不想做 KTV 工程，快速导入一首本地音频直接播放
- **场景 E**：用户在制作过程中想对比原声和伴奏效果
- **场景 F**：用户想纯欣赏模式，不需要歌词滚动

### 功能设计

#### 8.6.1 播放器双模式

| 模式 | 说明 | 使用场景 |
|------|------|----------|
| **KTV 模式** | 伴奏 + 滚动歌词 + 原声对比 | 练歌、制作 |
| **纯播放模式** | 原声播放 + 可选歌词 + 标准音乐播放器 UI | 听歌、预览 |

#### 8.6.2 快速导入播放
- 不强制创建工程，可选择"仅播放"
- 支持从文件管理器直接打开音频文件（关联文件类型）
- 播放历史自动保存，方便下次快速访问

#### 8.6.3 播放源切换
在 PlayerScreen 中提供切换按钮：
- **原声带** - 使用导入的原始音频
- **伴奏** - 使用分离后的 instrumental（如果有）
- **人声** - 使用分离后的 vocals（用于校对）

#### 8.6.4 歌词显示开关
- 用户可以关闭歌词，获得纯净播放器体验
- 歌词开启时支持全局偏移调节
- 支持歌词字体大小调节

### UI 设计原则（Spotify-inspired）
- 大封面图居中，占屏幕 40-50% 高度
- 歌曲信息在封面下方，层级清晰
- 播放进度条使用 accent 绿色
- 播放/暂停按钮使用圆形悬浮设计
- 底部工具栏包含：歌词开关、播放源选择、更多选项

## 补充 Phase 1.5：跨端工程传输

### 目标
让用户能方便地将桌面端制作的 KTV 工程传输到手机端播放，无需云存储、无需数据线。

### 技术方案选型

| 方案 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| A. 本地网络共享 | 不依赖互联网，速度快，隐私好 | 需要在同一局域网 | **主方案** |
| B. 二维码 + 局域网 HTTP | 配置简单，扫码即连 | 需要稳定局域网 | **主方案** |
| C. iCloud/Google Drive 同步 | 自动化，跨网络 | 依赖第三方服务，隐私风险 | Phase 3 |
| D. 物理数据线 | 可靠，大文件友好 | 操作繁琐 | 不做 |

### 功能设计

#### 1.5.1 发送端（桌面）

**入口位置**：ProjectDetailScreen → "传输到手机" 按钮

**发送流程**：
1. 用户点击"传输到手机"
2. 系统启动本地 HTTP 服务器
3. 生成二维码（包含局域网 IP + 端口 + 工程 ID）
4. 显示二维码弹窗，等待手机扫描
5. 传输完成后自动关闭服务器

**安全机制**：
- 仅同一局域网可访问
- 传输使用一次性 token 验证
- 服务器 5 分钟无连接自动关闭

#### 1.5.2 接收端（手机）

**入口位置**：Dashboard → "接收工程" 按钮

**接收流程**：
1. 用户点击"接收工程"
2. 打开相机扫描二维码
3. 解析二维码获取桌面端地址
4. 建立连接，下载工程文件
5. 自动解压并导入到本地存储
6. 显示导入成功，可直接打开播放

**文件结构**：
传输时打包为 `.lyricforge` 压缩文件，包含：
```
project.lyricforge (zip)
├── manifest.json          # 工程元数据
├── audio/
│   ├── original.mp3       # 原声（可选）
│   ├── instrumental.mp3   # 伴奏
│   └── vocals.mp3         # 人声（可选）
└── lyrics/
    └── lyrics.json        # 歌词时间轴
```

#### 1.5.3 存储位置约定

| 平台 | 工程存储路径 | 说明 |
|------|-------------|------|
| Desktop | `~/LyricForge/Projects/{project_id}/` | 用户可配置 |
| iOS | `Documents/LyricForge/Projects/` | App 沙盒 |
| Android | `ExternalStorage/LyricForge/` | 用户可见 |

### UI 设计

**发送端弹窗**：
- 标题："传输到手机"
- 中央：大二维码（支持点击放大）
- 底部：IP 地址 + 端口文字显示
- 状态：等待连接 / 传输中 / 完成
- 取消按钮

**接收端页面**：
- 标题："接收工程"
- 中央：相机预览（二维码扫描）
- 手动输入：IP:Port 输入框（备用）
- 底部：传输进度条（接收时显示）

## 数据模型更新

### AudioAsset 扩展
```dart
class AudioAsset {
  final String? originalPath;      // 原声文件路径
  final String? instrumentalPath;  // 伴奏路径
  final String? vocalsPath;        // 人声路径
  final AudioSource defaultSource; // 默认播放源
}

enum AudioSource {
  original,      // 原声
  instrumental,  // 伴奏
  vocals,        // 人声
}
```

### ProjectManifest 扩展
```dart
class ProjectManifest {
  // ... 现有字段 ...
  
  final bool isQuickPlay;      // 是否快速播放模式（无完整工程）
  final DateTime? lastPlayed;  // 上次播放时间（用于历史记录）
  final PlayMode preferredMode; // 用户偏好模式
}

enum PlayMode {
  ktv,      // KTV 模式：伴奏 + 歌词
  music,    // 音乐模式：原声 + 可选歌词
}
```

## 验收标准补充

### 独立播放器
- [ ] 用户可以在 3 步内完成"导入音频 → 直接播放"
- [ ] 播放器支持原声/伴奏/人声三种源切换（有则显示，无则禁用）
- [ ] 歌词可以一键开关，开关状态记忆到工程
- [ ] 播放器 UI 符合 Spotify 设计语言（暗色、封面居中、控件清晰）

### 跨端传输
- [ ] 桌面端生成二维码到手机完成导入，总耗时 < 30 秒（100MB 工程）
- [ ] 同一 WiFi 下自动发现，无需手动输入 IP
- [ ] 传输过程显示进度百分比和预计剩余时间
- [ ] 传输完成后手机端可直接播放，无需额外配置

## 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 局域网发现不可靠 | 跨端体验差 | 备用方案：手动输入 IP + 二维码 |
| 音频文件过大（>500MB） | 传输慢，失败率高 | 压缩打包传输，支持断点续传 |
| 防火墙阻止端口 | 无法建立连接 | 自动端口选择 + 错误提示引导 |
| iOS 文件系统限制 | 无法直接访问文件 | 使用 App Group / Share Extension |
