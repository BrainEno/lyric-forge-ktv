## What I do

I help evaluate and implement UI changes so the product stays aligned with a Spotify-inspired playback-first design language.

## Design intent

The app should feel:
- dark-first
- media-centric
- polished and immersive
- clean and modern
- dense but not cluttered
- elegant in motion and spacing

## Preserve these qualities

- strong focus on artwork / current track / current project
- subtle layered surfaces instead of loud borders
- rounded cards and modern control surfaces
- clear active-state emphasis
- restrained but smooth animation
- excellent typography hierarchy
- playback-first interaction flow

## Never do this

- do not copy Spotify logos or assets
- do not reproduce Spotify text verbatim
- do not clone exact iconography
- do not create generic admin-dashboard looking pages
- do not overuse outlined boxes or noisy borders
- do not make dark UI muddy or low-contrast

## When to use me

Use this skill when:
- building player screens
- refining lyrics editor UI
- adjusting navigation / sidebars
- polishing dark mode
- reviewing spacing, typography, and control density

## Output expectations

When reviewing or proposing UI changes:
1. identify hierarchy problems
2. identify spacing / density problems
3. identify places where the screen stops feeling media-centric
4. propose concrete layout and style changes
5. keep the result implementation-friendly
